# 🤖 Иру — Отказоустойчивый ИИ-Ассистент

Иру — это продвинутый Telegram-бот, построенный на базе современных нейросетевых архитектур (Transformer). Ключевой особенностью проекта является **трехуровневая отказоустойчивая система**, гарантирующая стабильную работу ассистента даже при сбоях основных провайдеров моделей.

Бот способен вести осмысленный диалог, понимать голосовые сообщения и адаптироваться к стилю общения пользователя.

### 1. Используемые API

Для реализации проекта была использована комбинация из трех ключевых API:

1.  **Telegram Bot API**
    -   **Описание:** Это основной API для взаимодействия с мессенджером Telegram. Он позволяет боту получать сообщения от пользователей, отправлять ответы, обрабатывать медиафайлы (голосовые сообщения), а также использовать элементы UX, такие как статус «печатает» и реакции на сообщения.

2.  **Hugging Face Inference API**
    -   **Описание:** Облачный сервис от платформы Hugging Face, предоставляющий доступ к тысячам предварительно обученных моделей. В нашем проекте он используется как шлюз для доступа к **основной, наиболее качественной модели ИИ**.

3.  **OpenRouter API**
    -   **Описание:** Это API-агрегатор, который предоставляет единый интерфейс для доступа к моделям от десятков различных провайдеров (включая Google, MistralAI и др.). Мы используем его для реализации **двух эшелонов резервирования**, что обеспечивает высокую надежность бота.

### 2. Базовая модель ИИ

В качестве основной («базовой») модели ИИ выступает **`Qwen/Qwen2.5-72B-Instruct`**.

-   **Описание:** Это сверхбольшая языковая модель (72 миллиарда параметров) от компании Alibaba. Она была выбрана за высокое качество генерации текста, отличное понимание русского языка и способность поддерживать сложные контекстуальные диалоги.

---

### 3. Модель трансформера и решаемые задачи

Весь проект построен на основе нейросетевой архитектуры **Transformer**, которая использует механизм внимания (Self-Attention) для обработки текста.

-   **Используемая модель:** `Qwen/Qwen2.5-72B-Instruct` (и резервные модели, также основанные на Transformer).

-   **Решаемые задачи:**

    1.  **Генерация ответов (Text Generation):** Основная задача. Модель генерирует осмысленные и релевантные ответы на основе запроса пользователя и истории диалога.

    2.  **Обработка контекста (Context Processing):** Модель анализирует историю последних 10 сообщений («скользящее окно»), чтобы поддерживать связный и логичный разговор.

    3.  **Классификация текста (Text Classification):** Модель неявно используется для определения эмоционального окраса сообщения пользователя, что позволяет боту ставить адекватные реакции-эмодзи.
